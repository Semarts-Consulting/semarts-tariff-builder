# MVP Demo Path Check

Date: 2026-06-22

Status: accepted by user on 2026-06-22.

Purpose: provide a lightweight walkthrough path for validating the internal MVP candidate without expanding scope.

## Scope

This check follows the agreed MVP path:

1. Inputs.
2. Allocation methods.
3. Tariff calculation.
4. Audit trace.
5. Output schedule.
6. Revenue recovery reconciliation.
7. Report view.

Formal export DTOs, advanced report generation, additional tariff scenarios, and UI polish are out of scope for this check.

- PR #7 is merged to `main`.

## Acceptance Questions

The demo path can support internal MVP candidate review if the reviewer can answer yes to each question:

- Can the reviewer identify the customer groups and demand/consumption inputs?
- Can the reviewer identify the recoverable cost base?
- Can the reviewer see which allocation method applies to each cost pool?
- Can the reviewer see fixed, energy, demand, and pass-through tariff outputs?
- Can the reviewer reconcile allocated outputs to the recoverable cost base?
- Can the reviewer see over/under recovery clearly?
- Can the reviewer inspect calculation steps in the audit trace?
- Can the reviewer identify warnings or limitations before approval?
